More About Javascript

What really is javascript?

JavaScript is a highly abstracted programing language

An Abstraction is a way of

  • Hiding the implementation details
  • Showing only the functionality to the users.
Example of high abstraction is Atm booth.

High Abstraction in javascript

  1. We don't have to handle resource management and memory allocation.
  2. We don't have to handle garbage Collection
  3. Reduces details so that developers can focus on logic better.
  4. Improves understandibility as well as maintainability of the code

Garbage Collection

  • Javascript automatically collects unused data.
  • Free the memory with the help of an algorithm called 'Mark-and sweep'
  • The garbage collector goes through the roots, marking(remembering) them on its way.
  • It moves on to the references and marks them as well.
  • The cycle continues until the garbage collector visits all the roots and references.
  • The garbage collector removes all the objects, except the marked ones.

JIT Compiled language

JavaScript is a Just-In-Time(JIT) comiled language.

  1. JavaScript is not a purely interpreted.
  2. Modern JavaScript is JIT compiled.
  3. Just-In-Time compiler convert the entire code into machine code and execute them immidiately.

Multi-Paradigm programming language

JavaScript is a Multi-Paradigm programming language.

  1. Procedural Programming
    • involves writing down instructions
    • tells the computer what it should do step-by-step
  2. Object oriented Programming
    • Modeling a system as a collection of objects.
    • Object contain both functions(or methods) and data.
  3. Functional Programming
    • an approach to software development that uses pure function.
    • To create maintainable software

proto-typed based programming language

Javascript is a proto-typed based programming language

  1. In javascript, everything(including function, array, objects) is object except the primitive data.
  2. a prototypical object is an object used as a template from which to get the initial properties for a new object.
  3. Proto-type is blue print.

Dynamically typed language

Javascript is a dynamically typed programming language.

  1. When you declare a variable, you do not need to specify what type this variable is.
  2. JavaScript engine infers what type this variable is based on the value assigned to at run time.
  3. As Javascript determines the type at runtime, we can re-declare the type.
