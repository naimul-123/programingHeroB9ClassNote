What is DNS?

  1. The domain Name System(DNS) is the phonebook of the Internet.
  2. Humans access information online through domain names, like nytimes.com or espan.com.
  3. Web browsers interact through internet protocol(IP) address.
  4. DNS transletes domain names to IP addresses so browsers can load Internet resources.

What is HTTP?:

  1. HTTP stands for Hyper Text Transfer Protocol.
  2. It is a protocol for fetching resources such as HTML documents.
  3. It is the foundation of any data exchange on the web and it is a client-server protocol which means requests are initiated by the recipient, usually the web browser.

HTTP Layer:

HTTP is in the Application layer of the internet protocol suit model and in the session layer of the OSI Model.

7 layers of the OSI Model

Application

  • End user layer
  • HTTP, FTP, IRC, SSH, DNS

Presentation

  • Syntax layer
  • SSL, SSH, IMAP, FTP, MPEG, JPEG

Session

  • Synch & send to port
  • API's, Sockets, WinSock

Transport

  • End to end connections
  • TCP, UDP

Network

  • Packets
  • IP, ICMP, IPSec, IGMP

Data Link

  • Farmes
  • Ethernet, PPP, Switch, Bridge

Physical

  • Physical Structure
  • Coax, Fiber, Wireless, Hubs, Repeators

HTTP vs HTTPS:

HTTP
  • HTTP stands for 'Hyper Text Transfer Protocol
  • HTTP works at the application layer
  • The default port number is 80, for communication
  • No encryption is present in HTTP websites
HTTPS
  • HTTPS stands for 'Hyper Text Transfer Protocol Secure
  • HTTPS works at the transport layer
  • The default port number is 443, for communication
  • Both encryption and decryption exist on HTTPS websites

Javascript Event:

HTML events are "things" that happen to HTML elements. When Javascript is used in HTML pages, Javascript can "react" on these events. For example:

  • The user select, clicks or hovers the cursor over a certain element.
  • The user chooses a key on the keyboard.
  • The user resizes or closes the browser window.
  • A web page finishes loading
  • A form is submitted
  • A video is played, paused or finishes.
  • An error occurs.

  2. onchange

  6. onblur

Event Bubbling & event capturing:

event bubbling is propagating an event from the lowest to the upword.

We can understand event bubbling as a sequence of calling the event handlers when one element is nested in another element and both the elements have registered listeners for the same event. So its ancestors on the way to top to bottom, calling is performed .

But when we capture event it starts from top element to the target element.

if we don't want propagate, we can do this with-> event.stopPropagation()

Event Delegation:

Event delegation is basically a pattern to handle events efficiently. Instead of adding an event listener to each and every similar element, we can add an event listener to a parent element and call an event on a particular target using the .target property of the event object.
