What really is javascript?

JavaScript is a highly abstracted programing language

An Abstraction is a way of

  • Hiding the implementation details
  • Showing only the functionality to the users.
Example of high abstraction is Atm booth.

High Abstraction in javascript

  1. We don't have to handle resource management and memory allocation.
  2. We don't have to handle garbage Collection
  3. Reduces details so that developers can focus on logic better.
  4. Improves understandibility as well as maintainability of the code

Garbage Collection

  • Javascript automatically collects unused data.
  • Free the memory with the help of an algorithm called 'Mark-and sweep'
  • The garbage collector goes through the roots, marking(remembering) them on its way.
  • It moves on to the references and marks them as well.
  • The cycle continues until the garbage collector visits all the roots and references.
  • The garbage collector removes all the objects, except the marked ones.

JIT Compiled language

JavaScript is a Just-In-Time(JIT) comiled language.

  1. JavaScript is not a purely interpreted.
  2. Modern JavaScript is JIT compiled.
  3. Just-In-Time compiler convert the entire code into machine code and execute them immidiately.

Multi-Paradigm programming language

JavaScript is a Multi-Paradigm programming language.

  1. Procedural Programming
    • involves writing down instructions
    • tells the computer what it should do step-by-step
  2. Object oriented Programming
    • Modeling a system as a collection of objects.
    • Object contain both functions(or methods) and data.
  3. Functional Programming
    • an approach to software development that uses pure function.
    • To create maintainable software

proto-typed based programming language

Javascript is a proto-typed based programming language

  1. In javascript, everything(including function, array, objects) is object except the primitive data.
  2. a prototypical object is an object used as a template from which to get the initial properties for a new object.
  3. Proto-type is blue print.

Dynamically typed language

Javascript is a dynamically typed programming language.

  1. When you declare a variable, you do not need to specify what type this variable is.
  2. JavaScript engine infers what type this variable is based on the value assigned to at run time.
  3. As Javascript determines the type at runtime, we can re-declare the type.

Execution Context and Call Stack

কোন কোড কোথায় রান হবে এটা নিয়ে আলোচনা করে।

  1. An execution context is an environment.
  2. Inside the execution context a piece of javascript code gets executed.
  3. Variables, parameters and other information related to the piece of code get stored in the execution context.

Execution Context

There are two kinds of execution context in javascript.

  • Global Execution Context(GEC)
  • Function Execution Context(FEC)

Global Execution Context

Whenever the javascript engine receives a script file, it first creates a default execution context known as the global execution context(GEC)

  • GEC is the base/default execution context.
  • All javascript code that is not inside of a function gets executed.
  • For every javascript file, there can only be one GEC.

Functional Execution Context

Whenever a function is called, the javascript engine create a different type of execution context known as a function execution context (FEC)

  • Every time a function is called, a new execution context is created for that function.
  • Each function has its own execution context.
  • Since every function call gets its own FEC, there can be more than one FEC in the run-time of a script.

Phase

The execution context is created in two phase:

  1. Creation phase
  2. Execution phase

Creation Phase

Creation phase is the phase in which the js engine has called a function but its execution has not started.

  1. Js engine is in the compilation phase
  2. It just scans over the function code to compile the code.
  3. it dose not execute any code.

Execution Phase

During the execution phase, the javascript engine executes the code line by line.

  1. Assigns the value to variables and
  2. Executes the function calls.

Call stack

The call stack is used by javascript to keep track of mulitple function calls. in order to manage the execution context, the javascript engine uses a call stack.

Single-threaded Asynchronous vs Synchronous

By default javascript is single-threaded.

Single-threaded means

  • only one statement is executed at a time.
  • one call stack
  • runs code line by line
  • Must finish executing a piece of code before moving onto the next.

Promise

The promise object represents the eventual completion(or failur) of an asynchronous operation and its resulting value. it receives a callback function as parameter. The callback function has two parameter. 1. resolve and 2. reject.
A promise is in one of these states:

  • Pending initial state, neither fulfilled nor rejected.
  • Resolve the operation was completed successfully.
  • Reject the operation failed.

Fetch API

  • The fetch() method starts the process of fetching a resource from a server.
  • The fetch() method returns a promise that resolves th a response object.
  • A fetch() method only rejects when a network error is encountered.

Advantage of Async/Await

  • async/await is a special syntax to work with promises in a more comfortable way.
  • It's surprisingly easy to understaind and use.
  • The await keyword can only be used inside an async function.
  • The await keyword makes t he function pause the execution and
  • wait for a resolved promise before it continues.

Example of async function:
// regular function
async function loadData (){
const res = await fetch('url here');
const data = await res.json();
console.log(data);
}

// ES6 function
const loadData = async()=>{
const res = await fetch('url here');
const data = await res.json();
console.log(data);
}
